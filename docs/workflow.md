# JD-to-Portfolio Workflow

This workflow converts a job description and a structured experience knowledge base into tailored resume and portfolio outputs.

## 1. Intake

Collect:

- Job description text.
- Target company/product context.
- Candidate experience cards.
- Optional reference websites.
- Optional visual assets that are safe to publish.

Do not collect:

- Private phone numbers or emails.
- Internal company screenshots.
- Raw user data.
- Proprietary table names, server names, or internal tools.

## 2. Experience Knowledge Base

Write every experience as a card:

- Context.
- Problem.
- Candidate role.
- Decisions made.
- Workflow or product design.
- Evidence.
- Measurable result, if available.
- Public-safe phrasing.

The goal is not to store every detail. The goal is to make future JD matching fast and accurate.

## 3. JD Analysis

For each JD, extract:

- Explicit responsibilities.
- Explicit requirements.
- Hidden requirements.
- Risk points.
- Keywords.
- Best-matching experience cards.
- Weak or missing evidence.

## 4. Resume Strategy

Choose one of three strategies:

- Strong match: most aggressively aligned to the JD.
- Balanced: preserves broader career context.
- Differentiated: emphasizes unusual strengths and founder/product signals.

## 5. Portfolio Page Strategy

The HTML version should not be a web copy of the paper resume.

It should include:

- A clear positioning statement.
- Why this candidate fits this JD.
- Deep project sections.
- Supporting experience.
- Personal texture.
- Safe visual materials.

## 6. QA

Before final delivery:

- Open the HTML locally.
- Check desktop and mobile screenshots.
- Check copy length.
- Check color consistency.
- Check privacy boundaries.
- Check that the page reads like a portfolio, not a pasted resume.
